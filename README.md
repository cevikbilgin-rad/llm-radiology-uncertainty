# Ordinal Uncertainty Grading in Chest Radiograph Reports Using Large Language Models

**A REFINE-Aligned Feasibility and Methodology Study**  
Manuscript: `DIR-2026-192` — Diagnostic and Interventional Radiology resubmission package

---

## Important contamination notice

This repository accompanies a Path-2 resubmission framed as a **feasibility and methodology study**, not as a definitive generalization benchmark. The OpenI Indiana University Chest X-ray dataset was publicly released in 2015-2016 and may have appeared in the pretraining corpora of contemporary frontier LLMs. Therefore, all performance estimates should be interpreted as **upper-bound feasibility estimates** rather than model-unseen validation results.

The primary contribution is the REFINE-aligned annotation protocol, ordinal scoring framework, sensitivity analyses, and reproducible statistical workflow for evaluating linguistic diagnostic uncertainty in chest radiograph reports.

## Study design overview

- Dataset: OpenI Indiana University Chest X-ray reports
- Raw reports identified: 3,955
- Excluded: 28 reports missing Findings or Impression sections
- Eligible reports: 3,927
- Validation cohort: 100 reports selected by stratified random sampling, 25 reports per hedge-frequency stratum (0, 1, 2, >=3 hedge words)
- Discovery cohort: 3,827 reports used only for exploratory descriptive analysis of model-derived uncertainty distribution
- Manual reference standard: three independent radiology annotators with majority vote; complete three-way disagreement adjudicated by the attending radiologist
- Primary endpoint: Cohen quadratic weighted kappa vs multi-reader consensus, with 1,000 bootstrap iterations and seed=42

## Main results reported in the manuscript

| Method | Quadratic weighted kappa | 95% bootstrap CI | Conservative agreement label |
|---|---:|---:|---|
| Gemini 2.5 Pro | 0.814 | 0.739-0.872 | Substantial |
| Claude Opus 4.5 | 0.807 | 0.726-0.869 | Substantial |
| Lexicon baseline | 0.622 | 0.516-0.713 | Moderate |
| Inter-model agreement | 0.857 | 0.801-0.901 | Almost perfect |

Agreement categories follow Landis and Koch without hybrid labels. For model-to-consensus and lexicon-to-consensus rows, the displayed label uses the lower bound of the 95% confidence interval. Inter-model agreement is descriptive and uses the point estimate.

## Reviewer-requested sensitivity analyses

| Sensitivity analysis | Claude Opus 4.5 | Gemini 2.5 Pro |
|---|---:|---:|
| PGY-2/operator excluded; senior resident + attending reference standard | kappa=0.831 | kappa=0.832 |
| Complete three-way disagreement cases excluded (n=93) | kappa=0.816 | kappa=0.810 |
| PGY-2/operator excluded + complete-disagreement cases excluded | kappa=0.841 | kappa=0.829 |

Because the attending radiologist was the prespecified tie-breaker for senior-attending disagreements, the PGY-2/operator-excluded two-reader reference is equivalent by construction to the attending radiologist independent annotations. This sensitivity analysis is therefore interpreted transparently as an LLM-versus-attending analysis.

## LLM output interpretation

The LLM prompt returns two scoring fields:

```json
{
  "uncertainty_level": "none|low|moderate|high",
  "uncertainty_score": 0,
  "key_phrases": ["list of actual hedging phrases found"],
  "reasoning": "brief 1-sentence explanation"
}
```

Only `uncertainty_level` is used for the primary analysis. It is mapped to the 0-3 ordinal scale as follows:

| uncertainty_level | Primary ordinal score |
|---|---:|
| none | 0 |
| low | 1 |
| moderate | 2 |
| high | 3 |

`uncertainty_score` is collected only as a secondary descriptive 0-10 output and is not used in the primary analysis.

## Full pipeline usage where data access permits

```bash
# 1. Download OpenI reports and compute lexicon scores
python 01_data_preparation.py

# 2. Create validation and discovery cohorts
python 02_sampling.py

# 3. Run LLM inference on validation cohort
export ANTHROPIC_API_KEY="your-claude-key"
export GEMINI_API_KEY="your-gemini-key"
python 03_claude_api.py --input validation_set_100.xlsx --output claude_validation_100.json
python 04_gemini_api.py --input validation_set_100.xlsx --output gemini_validation_100.json

# 4. Construct manual reference standards from neutral annotation files, if available
python 05_consensus.py

# 5. Compute primary and sensitivity analyses
python 06_statistics.py

# 6. Check for verbatim fabricated phrase-level evidence
python 07_hallucination_check.py

# 7. Exploratory descriptive domain analysis only
python 08_clinical_domain_analysis.py --input discovery_set_3827_with_claude.xlsx

# 8. Stochasticity and prompt sensitivity analyses
python 09_stochasticity_test.py
python 10_prompt_sensitivity.py
```

## Safety and limitation notes

- No clinical workflow integration was performed.
- No prospective deployment was performed.
- The discovery cohort lacks an independent human reference standard.
- Phrase-level checks verify absence of verbatim fabricated evidence only; they do not prove absence of semantic/contextual misattribution.
- External validation on confirmed model-unseen institutional reports remains necessary before making claims about generalization performance.
